# cs-210-Cplusplus-Program

Summarize the project and what problem it was solving.
  The purpose of this project is to accept user inputs for their initial investment amount, monthly deposit, annual interest, number of years to invest, and output the calculated investment amount. At a development level, this project determines if students can create logic that meets the requirements of an investment banking application. 
  
What did you do particularly well?
  Per feedback on this assignment, I was able to develop an application that correctly validated the user input, the UI matched the assignment’s requirements, and the console output the expected results. 

Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?
  Per feedback on this assignment, I could further simplify the design of my code and have less files. Simplifying the design structure would allow future maintenance to be less time-consuming and more efficient. In a larger scale project, having too many files may also cause code compilation to take up a considerable amount of time, which may cause time-out issues. 

Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?
  To me, there wasn’t one single piece of code that was challenging to write, per se. With that said, the most challenging part of this assignment was troubleshooting to ensure that the console output matched the expected outcome, as documented in the sample input/output for the assignment. It took me several hours to go through each part of my code and look through Stack Overflow to figure out why my output did not match the sample. I will continue using Stack Overflow as a resource to solve these issues, as well as use the Visual Studio debugger tool when I run into similar issues in the future. 

What skills from this project will be particularly transferable to other projects or course work?
  The skills from this project that are transferable to other projects or course work are code and file structure modularization, proper naming and commenting conventions, and creating code that is transferable, readable, and reusable. 

How did you make this program maintainable, readable, and adaptable?
  This program is maintainable by modularizing functions, classes, etc. based on their intended purposes. Similarly, modularizing my code allows it to be readable and adaptable by other developers who may need to debug, make updates to the code, and/or add a new feature and ensure that it works with the existing code.
